# Melda-n-n-Maceralar-A-Personal-Learning-Journey
This is a personal, non-commercial RPG project created as a hands-on exercise to explore game development logic in C#. It’s a tribute to the classic top-down adventure games, built specifically to practice mechanics like state machines, UI synchronization, and enemy AI.

    Note: This project was developed for educational purposes to consolidate my knowledge of C# and game design patterns. While building it, I followed various tutorials and community resources to understand best practices.

The Experience

The game follows a simple yet complete loop:

    Starting Out: You begin in a village, interact with an NPC, and manage your resources (Health, Energy, Gold).

    Mechanics: Practice with Stamina-based Dashing and melee combat.

    Dungeon Crawling: Exploring a cave with locked-room triggers and a specialized upgrade shop.

    The Challenge: A final encounter with a 3-Phase Boss that changes attack patterns as the fight progresses.

 What I Practiced Here

    Logic Integration: Bringing together variables (Gold, HP, Energy) into a functional gameplay loop.

    Scene Transitions: Managing data and state while moving between the village and the dungeon.

    Problem Solving: Implementing levers, gates, and randomized loot drops.

    AI Patterns: Creating different behaviors for simple enemies (Slimes) versus complex enemies (Multi-phase Boss).

  Purpose of the Project

The goal wasn't to create a commercial product, but to see how different systems (Combat, UI, Save/Load, Movement) communicate with each other. It represents the "final boss" of my current learning stage before moving on to more complex architectures.
